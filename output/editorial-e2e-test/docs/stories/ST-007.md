# ST-007 — Implement authentication

**Story key:** `feature.authentication`

## Description

Add registration, login, logout and session management.

## Acceptance Criteria

- [ ] Users can register with email and password — POST /api/auth/register returns 201 on success, 400 on validation error, 409 on duplicate email
- [ ] Users can log in and receive a valid session — POST /api/auth/login returns 200 with session token on success, 401 on invalid credentials
- [ ] Users can log out and have their session invalidated — POST /api/auth/logout returns 200 on success
- [ ] Protected routes return 401 for unauthenticated requests
- [ ] Passwords are hashed before storage
- [ ] Payload Users collection has auth enabled
- [ ] Database migration is generated and executed for all schema changes (run `npm run db:migrate:create` then `npm run db:migrate`)

## Scope Boundaries

- Do NOT implement OAuth or social login in this story
- Do NOT implement password reset or email verification
- Do NOT implement MFA
- All migrations MUST be created in `src/lib/migrations/` using `npm run db:migrate:create` — do NOT create migration files in any other directory (e.g. `src/models/migrations/`)

## Expected Files

- `src/lib/auth.ts`
- `src/collections/Users.ts`
- `src/app/(auth)/login/page.tsx`

## Dependencies

- `bootstrap.backend`
- `bootstrap.frontend`

## Validation

- Run: `npm run build`
- Run: `npm test`
- Manual: Can register, log in, and access protected route
